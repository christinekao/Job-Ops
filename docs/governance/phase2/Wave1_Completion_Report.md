## Implementation Summary

### Effective CV Brief

- `isCvBriefUsable` requires selling points, must-show evidence, and a bullet plan.
- `resolveEffectiveCvBrief` reuses a persisted Brief only when usable and content-equivalent to the current derived Brief.
- `cvBriefIdentityHash` excludes `generatedAt` so metadata timestamps do not create false context drift.
- Legacy full-Brief/source hashes remain accepted by `cvStaleReasonForJob`.
- No persisted CV or canonical runtime snapshot was migrated.

### Evidence Priority

- `selectionDiagnostics` now returns selected objects in selected-ID order.
- Canonical downstream priority is effective Brief must-show IDs, then remaining selected evidence IDs.
- Writer projection, local repair candidates, GenerationContext, and Screening Gate use the same priority.
- `evidenceSelectionQualityDiagnostics` exposes coverage/quality allocation without changing the selection algorithm.

### Writer Context

- Writer receives a bounded projection instead of full career/evidence objects.
- Weak/Unsupported gaps and all selected evidence/skill/story mappings remain available.
- `GenerationContext` adds optional `writerContextHash` and `evidencePriorityIds`.
- Context ratios versus the previous JSON payload:
  - AI Evaluation JD: 0.4522 (54.8% reduction)
  - Power Platform JD: 0.4490 (55.1% reduction)

### Contact Ownership

- ADR-007 assigns optional structured contact ownership to `CareerProfile.contact`.
- Existing data without contact remains valid.
- Writer receives an empty contact object when no authoritative contact is populated.
- Completeness enforcement remains Wave 2 scope.
