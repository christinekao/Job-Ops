## Writer Input Rules

- Use only selected, source-grounded evidence.
- Use CV Brief as the plan.
- Use Screening Analysis as constraints.
- Translate or suppress internal terms.
- Do not force unsupported missing keywords.
- For repair, patch only failed checks in `fixContext`.
- First-pass output should aim to pass local review without relying on a second repair.
- Active Writer prompt construction is owned by `src/promptBuilders.ts`; `data/prompt_templates.json` is UX metadata/reference content and must not silently alter this runtime input.
- Writer uses `resolveEffectiveCvBrief`; it must not read a different persisted Brief from UI/hash/apply consumers.
- Writer evidence is serialized in effective evidence-priority order.
- Writer receives a bounded projection of career profile, Analysis, selected skills/domain/evidence/stories, and selection-quality diagnostics rather than full source objects.
- Optional structured contact is owned by `CareerProfile.contact`; missing contact remains explicit and is not inferred from raw source text.

## Known Gaps

- Prompt length and context truncation behavior were not measured in this phase.
- `npm run smoke:writer-input` now protects the meaningful prompt shape without freezing full wording: selected JD, Screening Analysis, CV Brief, selected evidence, JSON-only output, unsupported-claim controls, repair context, and green-area preservation.
- Writer input identity is represented by `GenerationContext.cvBriefHash` in current code; legacy persisted contexts without that field remain trace-limited.
- Phase 2 Wave 1 excludes `generatedAt` from Brief content identity, adds optional `writerContextHash`/`evidencePriorityIds`, and preserves legacy Brief/source hash compatibility.
- `npm run smoke:wave1-context` measures current Writer context projection at 45.2% and 44.9% of the legacy JSON payload for the two persisted JDs.

## Ownership

ADR-003 assigns runtime construction to `src/promptBuilders.ts`. This contract constrains its inputs and behavior but does not replace its implementation. Any future wiring from `data/prompt_templates.json` into production automation requires a separate approved implementation Task.
