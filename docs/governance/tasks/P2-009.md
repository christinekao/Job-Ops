## Title

Enforce Approved Contact Completeness at Apply and Export Boundary

## Objective

Ensure generated CVs cannot silently pass apply/export readiness with missing required contact fields when authoritative profile contact exists or is required.

- `docs/governance/phase2/QUALITY_SCORECARD.md`: HR Readability and Export Readiness penalize missing contact.
- `CV_Manager_React/src/domain/screeningCvOutput.ts`: apply validation owner.
- `CV_Manager_React/src/domain/screeningReview.ts`: export readiness owner.
- `CV_Manager_React/scripts/smoke-writer-output.mjs` and `smoke-export-readiness.mjs`: existing focused coverage.

Confidence: Confirmed

## Root Cause Addressed

RC-06: no canonical structured contact owner existed across source -> profile -> Writer -> export. Wave 1 established ownership; this task enforces the boundary.

## Scope

Single responsibility: enforce missing-contact behavior at apply/export boundaries without hard-coded fallback contact data.

In scope:

- Missing required email fixture.
- Apply/export validation rule using existing structured `CareerProfile.contact` ownership.
- No inference from historical source artifacts.

Out of scope:

- Migrating canonical runtime data.
- Editing profile UI.
- Hard-coding candidate contact values.
- Prompt wording.
