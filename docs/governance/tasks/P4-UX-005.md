## Acceptance Criteria

- [x] Exactly one primary CTA is visible at a time.
- [x] Primary CTA names the actual destination when a target is known.
- [x] First unresolved blocker is shown as `Your Next Step`.
- [x] Remaining blockers are ordered and not duplicated as a second full blocker list.
- [x] Pending blockers are not styled as completed.
- [x] Completed blockers use completed styling only.
- [x] Generic `Jump to Fix` is replaced by target-specific copy when possible.
- [x] Unknown targets show a manual-review fallback.
- [x] Internal reviewer terms are hidden from primary UI and retained only under Advanced Details.
- [x] Export status is shown once.
- [x] Export CTA appears when blockers reach zero.
- [x] View-only navigation does not alter workflow state.
- [x] No domain behavior changes.
- [x] No hidden AI invocation occurs.
- [x] Required validation passes.

## Completion

Completed 2026-07-13.

- Simplified Reviewer / Repair / Export presentation into Overall Status, Your Next Step, Remaining Issues, Repair Progress, Readiness and Export, and Advanced Details.
- Removed the separate visible Repair Plan + Primary CTA stack from the reviewer/export screen to avoid competing primary actions.
- Replaced generic `Jump to Fix` with target-specific CTA labels where a guided target is known: `Jump to Email`, `Jump to Summary`, `Jump to Bullet 1`, and `Export CV`.
- Unknown targets show an explicit manual-review fallback and disabled `Review Manual Decision` CTA.
- Browser Product Acceptance verifies a first-time user can follow the next step, reach the correct field, reduce blockers, and unlock export using fixtures only.
- Runtime prompts, review rules, repair rules, export decision rules, runtime data, persistence, and AI invocation behavior were not modified.
- Phase 5 was not started.
