## Acceptance Criteria

1. Summary role-fit blocker maps to targeted summary regeneration.
2. Weak achievements map to proposal when exact bullet target exists.
3. Weak achievements map to targeted bullet regeneration when target is broad.
4. External wording maps to proposal or targeted regeneration.
5. Missing email maps to human input when no trusted value exists.
6. Trusted email maps to safe local repair.
7. Duplicate email/contact blockers consolidate into one user task.
8. Targeted regeneration changes only allowed zones.
9. Preserved zones remain structurally equivalent.
10. Regenerated content keeps valid EvidenceCard traceability.
11. Unsupported claims are not introduced.
12. Validation failure preserves the current CV.
13. Successful regeneration creates a new CV version/hash.
14. Stale CV context rejects regeneration.
15. Effective-brief change rejects stale regeneration.
16. Affected review families rerun.
17. Unrelated passed checks remain passed.
18. Blocker count updates.
19. CTA advances to the next repair route.
20. Export appears when blockers clear.
21. No hidden AI call occurs.
22. Deterministic fixtures require no API credentials.
