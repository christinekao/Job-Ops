## Confirmed Root-Cause Trace

| Layer | Current behavior | Root cause |
|---|---|---|
| Click feedback | Parent lifecycle changes to running | Status renders only in the upper Your Next Step area, not beside a lower blocker-card click. |
| Runtime/validation | Actual automation and validation run | The UI returns to idle in `finally`; no persistent targeted terminal state exists. |
| Result | Generic `reviewerActionResult` remains elsewhere | No-diff copy is separated from the clicked action and reads like an inactive/general result. |
| Blocker refresh | Blocker correctly remains unresolved | Repair action is re-derived only from unchanged Orchestrator route. |
| CTA refresh | Same targeted regeneration returns | No attempt key/outcome participates in CTA derivation. |
| Retry | Static Retry/Edit buttons | Buttons have no handlers, no retry count, and no token-use warning. |

## Acceptance Criteria

1. Click feedback is immediate and visible beside both targeted action locations.
2. Running and validating stages remain visibly non-idle and explain protected/validated areas.
3. Duplicate clicks are rejected.
4. Success and no-diff results remain visible until another action or material context change.
5. No-diff uses a context identity containing blocker, strategy, target zones, CV hash, Brief hash, and evidence-context hash.
6. No-diff remains terminal for the same unchanged context and keeps the blocker unresolved.
7. The same regeneration is not the primary next action after no-diff.
8. A supported manual alternative becomes primary; retry is secondary, explicit, warns about token use, and records attempt count.
9. Same-context dispatch is blocked unless explicit Retry is used.
10. CV/Brief/evidence/blocker changes clear the active terminal state; timestamp-only changes do not.
11. No contradictory `AI updated - No CV content changed` or `Review Changes` no-diff copy remains.
12. No hidden AI call, architecture redesign, rule change, runtime-data mutation, additional READY task, or Phase 5 work is introduced.
