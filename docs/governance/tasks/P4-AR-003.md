## Acceptance Criteria

1. Executor consumes only non-stale Repair Orchestrator summaries.
2. Executor requires exact CV version id and content hash match.
3. Executor mutates only allowed zones for each safe-auto classification.
4. Executor preserves prohibited zones.
5. Executor creates a new CV version when a safe repair changes content.
6. Executor returns explicit execution result for success, stale, duplicate, and no-diff cases.
7. Executor prevents duplicate execution on the same CV hash and repair plan.
8. Executor preserves bullet evidence IDs and never invents facts, metrics, ownership, skills, or experience.
9. Deterministic tests cover two safe-auto repairs, stale orchestrator output, duplicate repair attempt, and no-content-diff repair.
