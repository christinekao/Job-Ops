# P4-AR-004 - Batch Proposal Review and Application

Status: DONE

## Objective

Implement batch review/application for accepted approval-required AI repair proposals.

This task must apply only accepted, non-stale proposals and create one new CV version for the accepted batch where safe.

## Dependencies

- P4-AR-003: DONE

## Forbidden Files

- Runtime prompts
- Runtime data
- Evidence data
- Review rules
- Safe repair executor behavior beyond integration
- Bounded repair loop
- Human decision application
- Export decision logic

## Acceptance Criteria

1. Proposal batch displays current vs suggested content, reason, evidence preserved, and expected impact.
2. Accept, reject, and edit-manually states are represented deterministically.
3. Accepted proposals apply in one CV version where safe.
4. Rejected proposals do not mutate CV content.
5. Stale proposals cannot execute.
6. Proposal application preserves evidence IDs and does not invent facts.
7. Export readiness is not changed by this task.
