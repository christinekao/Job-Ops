# P4-AR-005 - Bounded Repair Session

Status: DONE

## Objective

Implement a bounded repair session coordinator that runs only after explicit user action and stops safely.

The coordinator may sequence safe repair execution, accepted proposal application, scoped review refresh, and orchestrator refresh. It must never invoke hidden AI and must never loop indefinitely.

## Acceptance Criteria

1. Session starts only from explicit user action.
2. Default max iteration count is 3.
3. Session stops on no blockers, only human/unsupported blockers, no diff, repeated blocker set, max loop, budget stop, or unsafe stop.
4. Review refresh is scoped to affected review families where provided.
5. Orchestrator classification refresh happens after applied changes.
6. Same blocker set repetition is detected.
7. Deterministic tests cover convergence, partial convergence, repeated blockers, no-content-diff, and max loop reached.
