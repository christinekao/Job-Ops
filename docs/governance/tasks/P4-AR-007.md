# P4-AR-007 - Repair Proposal UI Integration and Disabled CTA Fix

Status: DONE

## Objective

Connect the existing P4-AR-001 through P4-AR-006 repair proposal and batch-application capabilities to the real Screening Lab UI.

This task fixes disabled or non-functional proposal CTAs, renders actual proposal cards, supports accept/reject decisions, applies accepted proposal batches through the existing CV save boundary, and keeps no-AI deterministic fixtures intact.

## Confirmed Root Cause

Observed Problem:
`Review N AI Suggestions` did not reliably open actual proposal content or lead to proposal application.

Immediate Cause:
`ExportDecisionPanel` selected only `proposals[0]` and rendered one `activeProposal`; it had no proposal batch review state or apply CTA.

Upstream Cause:
P4-AR-004 batch proposal application existed in the domain layer but was not wired to the production proposal UI.

Structural Root Cause:
The P4-AR-001 preview UI and P4-AR-004 application domain were never connected through the Screening Lab UI/action boundary.

## Acceptance Criteria

1. `Review N AI Suggestions` is enabled when approval-required candidates exist, current CV version/hash match, no conflicting operation is running, proposal generation is supported, and a handler exists.
2. Disabled proposal CTA shows an exact reason.
3. Clicking proposal CTA dispatches exactly one explicit proposal-generation action.
4. Proposal running state appears immediately.
5. Proposal results normalize into the existing proposal contract.
6. Invalid proposal results show explicit error and do not modify CV.
7. Proposal cards show current content, AI suggestion, why, expected impact, evidence safety, and decision buttons.
8. Accept/reject state updates counts.
9. `Apply N Accepted Changes` enables only when accepted non-stale proposals exist.
10. Applying accepted proposals creates one new CV version and content hash.
11. Rejected proposals do not mutate CV.
12. Review refresh is requested after apply.
13. Blocker count refreshes after apply.
14. Export CTA appears when blockers clear.
15. No-apply state does not show `AI did not update the CV`.
16. Stale proposals cannot be applied.
17. No-diff apply returns explicit result.
18. No hidden AI call occurs.
19. Fixture mode remains deterministic.
20. Existing P4-AR and Phase 4 tests remain passing.
