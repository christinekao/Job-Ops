## Confirmed Root Cause

The proposal UI inferred generation readiness from existing proposal objects and reused one state for multiple meanings. This allowed a contradictory state where approval-required candidates existed but the UI could still show `No valid AI suggestions were produced for the current CV.` before generation had actually run.

## Acceptance Criteria

1. Candidates with no generated proposals produce `ready`.
2. `ready` shows enabled `Generate N AI Suggestions`.
3. `ready` does not show the empty-result message.
4. Clicking Generate dispatches exactly one proposal-generation command.
5. Running state appears immediately.
6. Running CTA is disabled.
7. Successful generation transitions to `success`.
8. Success shows enabled `Review N AI Suggestions`.
9. Empty generation transitions to `empty`.
10. Empty shows `Retry AI Suggestions`.
11. Error shows a precise user-safe reason.
12. Stale proposals show `Generate New AI Suggestions`.
13. Timestamp-only changes do not create stale state.
14. Content-hash changes do create stale state.
15. Real-AI mode does not incorrectly disable Generate.
16. Proposal-candidate count and generated-proposal count are separate.
17. No CV mutation occurs during generation.
18. No proposal application occurs automatically.
19. No hidden AI invocation occurs.
20. Existing P4-AR and Phase 4 tests remain passing.
