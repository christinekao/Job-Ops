## Route Contract

Every blocker must resolve to exactly one route:

- `safe-auto`
- `approval-required`
- `human-decision`
- `unsupported`

UI must not independently decide the route.

## Classification Rules

Safe automatic repair candidate:

- exact duplicate removal
- deterministic formatting cleanup
- known internal terminology replacement when mapping is explicit
- missing email when a trusted profile email already exists
- deterministic ordering or normalization
- changes already supported by existing safe local repair boundaries

Approval required:

- summary rewrite
- work-bullet rewrite
- wording improvements
- business-impact reframing
- external resume wording
- evidence-preserving phrasing changes

Human decision:

- career positioning
- choosing between role identities
- adding unsupported experience
- deciding whether a claim is true
- choosing omitted or uncertain achievements
- evidence conflicts
- personal preference or strategic trade-off

Unsupported:

- no reliable edit target
- required evidence missing
- blocker cannot be safely repaired
- repository contracts do not authorize a repair path
