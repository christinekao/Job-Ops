# P4-VAL-001 - Scoped Validation for Targeted Regeneration

Status: DONE

## Objective

Replace the full-CV pre-apply gate in targeted regeneration with a scoped, patch-based validation boundary. A valid target-zone patch may apply while unrelated pre-existing global blockers remain authoritative for review/export.

## Selected Execution

- AI: Codex
- Model: GPT-5
- Reasoning: 中

## Validator Responsibility Matrix

| Validator | Current Scope | Correct Scope During Targeted Regeneration | Blocking? | Remaining Global Issue? |
|---|---|---|---|---|
| Runtime response normalization / request identity | Whole response and command context | Contract/stale gate | Yes | No |
| `validateScreeningCvOutput` required fields | Whole normalized candidate before patch | Target fields block; unchanged out-of-target fields are global issues | Target/new only | Yes |
| EvidenceCard namespace and traceability | Every candidate bullet | Changed target bullets block; unchanged bullet failures remain global | Target/new only | Yes |
| Unsupported visible claims | Post-candidate review plus structured unsupported terms | New claims in changed target content block | Yes | Pre-existing unrelated claims remain |
| Duplicate Summary/bullets | Whole normalized candidate | New duplicates involving changed target content block; unchanged outside duplicates remain global | Target/new only | Yes |
| Target-zone mutation boundary | Extracts allowed fields and currently ignores raw out-of-zone differences | Raw prohibited-zone mutation is a preserved-zone failure | Yes | No |
| Freshness checks | CV version/hash and Brief; evidence context in command identity | Contract/stale gate | Yes | No |
| Review and export evaluation | Whole saved CV after patch | Global release/export gate after scoped apply | No for local apply; yes for export | Yes |

## Acceptance Criteria

1. Targeted AI output is extracted into an authorized patch before apply-gate validation.
2. Stale/contract, target-zone, preserved-zone, pre-existing global, and new global issues are separated.
3. Pre-existing unrelated blockers never reject an otherwise valid target patch.
4. New target/global failures and any raw prohibited-zone mutation block application.
5. Non-target zones are preserved byte-for-byte at the domain zone boundary.
6. Successful patches use the existing CV save/version/hash, review, Repair Orchestrator, CTA, and export refresh boundaries.
7. Remaining global blockers stay visible and remain authoritative for export.
8. Diagnostics explain true apply-gate failures and list pre-existing global issues separately.
9. No prompt, review rule, export semantic, routing, evidence rule, persistence, or runtime-data change occurs.
