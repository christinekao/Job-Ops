# P2-010 - Separate Application-Fit Gaps from Visible-CV Integrity

Status: DONE

## Task ID

## Title

Separate Application-Fit Gaps from Visible-CV Integrity

## Objective

Stop treating honest omitted JD gaps as visible-CV integrity failures while still reporting application-fit risk.

## Repository Evidence

- `docs/governance/phase2/ROOT_CAUSE_ANALYSIS.md` RC-07: reviewer evaluates JD gaps as CV failures.
- `docs/governance/phase2/CV_PIPELINE_AUDIT.md`: reviewer can fail on analysis-level unsupported/weak gaps even when not visible in CV claims.
- `docs/governance/phase2/IMPLEMENTATION_PRIORITY_MATRIX.md` rank 7: Reviewer fails honest CVs for JD gaps.
- `CV_Manager_React/src/domain/screeningReview.ts`: `reviewerPass` owner.
- `CV_Manager_React/scripts/smoke-reviewer-export-gate.mjs`: existing reviewer/export smoke coverage.

Confidence: Confirmed

## Root Cause Addressed

RC-07: application-fit risk and visible-CV claim integrity are combined in one reviewer check.

## Scope

Single responsibility: separate visible overclaim/integrity checks from fit-gap reporting.

In scope:

- Fixture where an unsupported JD gap is honestly omitted from the visible CV.
- Reviewer result keeps fit risk visible without marking the CV claim integrity as failed.
- No lowering of unsupported visible claim detection.

Out of scope:

- Unknown keyword support handling covered by P2-011.
- Manager-impact proxy replacement.
- Repair routing.
- Prompt wording.
